# 02 — SQL and Databases

## Purpose

This folder contains labs for SQL, relational databases, operational reporting, data quality, and database-backed evidence.

The labs should help develop the ability to query data, reason about records, explain findings clearly, and turn database work into interview and portfolio evidence.

## Topic Focus

Labs in this folder may cover:

* SELECT queries
* filtering and sorting
* joins
* aggregation
* grouping and trend analysis
* relational schema design
* data quality checks
* duplicate and inconsistent record investigation
* database-backed support reporting
* indexing and query performance basics
* backup and restore concepts
* communicating database findings

## Mandatory Career Layers

Every SQL and database lab must also include:

* structured analytical thinking
* clear explanation of findings
* manager or stakeholder communication
* recruiter-facing summary
* interview explanation
* personal reflection on accuracy and evidence quality

## Example Lab Folder Names

```text
lab-01-query-support-ticket-data/
lab-02-join-users-devices-and-tickets/
lab-03-incident-trend-analysis/
lab-04-database-quality-review/
```

## Evidence Standard

SQL evidence should include safe sample data, query text, result summaries, row counts, assumptions, and limitations. Do not commit real company data, confidential exports, private user records, or screenshots containing sensitive information.

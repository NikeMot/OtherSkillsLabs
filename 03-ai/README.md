# 03 — AI

## Purpose

This folder contains labs for AI, LLMs, prompt engineering, AI-assisted operations, retrieval workflows, and evaluation.

The labs should develop practical AI capability without exaggerating what AI can do. The focus is useful, safe, explainable AI work for support, infrastructure, security, and operations contexts.

## Topic Focus

Labs in this folder may cover:

* prompt engineering for technical work
* AI-assisted ticket triage
* log and incident summarisation
* retrieval-augmented generation
* evaluation datasets
* hallucination checks
* AI workflow design
* privacy and data handling
* human approval points
* operational usefulness and limitations

## Mandatory Career Layers

Every AI lab must also include:

* failure-mode thinking
* clear explanation of AI limitations
* safe communication to managers or peers
* recruiter-facing summary
* interview explanation
* personal reflection on responsible use

## Example Lab Folder Names

```text
lab-01-ai-assisted-ticket-triage/
lab-02-prompt-engineering-for-support/
lab-03-runbook-retrieval-workflow/
lab-04-ai-output-evaluation/
```

## Evidence Standard

AI evidence should include test inputs, expected outputs, actual outputs, pass or fail criteria, and limitations. Do not use private company data, confidential tickets, real customer data, or sensitive logs.

# 03 — Data Engineering

## Purpose

This folder contains labs for SQL, databases, data modelling, pipelines, data quality, operational reporting, and small SRE-supporting applications.

The goal is to use data engineering to build useful evidence, models, and tools for SRE-style work: incident analysis, ticket trends, service health, reliability reporting, and operational decision support.

## Core Reference

A key reference for this topic is `Designing Data-Intensive Applications`, which is relevant because SRE work often depends on understanding reliable, scalable, and maintainable data systems.

Other supporting references may include data engineering, SQL, statistics, and Python data analysis books from the learning library.

## Topic Focus

Labs in this folder may cover:

* SQL for operational analysis
* relational modelling
* data quality checks
* CSV to database ingestion
* basic ETL pipelines
* operational reporting
* incident and ticket trend analysis
* indexes and query performance basics
* backup and restore concepts
* data-intensive system design concepts
* reliability, scalability, and maintainability
* building small SRE helper applications from data

## Mandatory Career Layers

Every data engineering lab must also include:

* analytical thinking
* clear explanation of findings
* stakeholder or manager communication
* recruiter-facing summary
* interview explanation
* personal reflection on accuracy, usefulness, and evidence quality

## Evidence Standard

Data engineering evidence should include safe sample data, schema notes, query text, result summaries, row counts, assumptions, limitations, and where relevant a small application or script.

Do not commit real company data, confidential exports, private user records, or screenshots containing sensitive information.

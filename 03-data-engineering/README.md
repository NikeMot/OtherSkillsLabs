# 03 — Data Engineering

## Purpose

This folder contains labs for SQL, databases, data modelling, pipelines, data quality, operational reporting, and small SRE-supporting applications.

The goal is to build junior data engineering evidence: pipeline development, ingestion, transformation, dataset modelling, SQL, Python, version control, documentation, testing, data quality checks, monitoring, incident investigation, and performance/cost awareness.

## Core Curriculum Rule

Use `Fundamentals of Data Engineering` as the foundation.

The first 11 labs are based on one chapter each from `Fundamentals of Data Engineering`.

The next 8 labs are drill labs that repeat and deepen the manual skills from the first 11 labs.

The final lab is a capstone project.

See `docs/data-engineering-curriculum-standard.md` for the full 20-lab curriculum.

## Manual-First AI Rule

Learn and complete the data engineering task manually first.

Use `Redefining Data Engineering with AI` only afterwards to learn how AI can assist with documentation, design review, test generation, quality checks, metadata work, explanation, and workflow acceleration.

Every AI-assisted note must separate:

* what was done manually
* what AI helped with afterwards
* what was verified by the learner
* what risks or limitations were found

## Topic Focus

Labs in this folder may cover:

* SQL for operational analysis
* relational modelling
* data quality checks
* source-system analysis
* CSV/API to database ingestion
* basic ETL/ELT pipelines
* transformation and curation
* operational reporting
* incident and ticket trend analysis
* indexes and query performance basics
* monitoring and incident investigation for data issues
* governance and access-control awareness
* data-intensive system design concepts
* reliability, scalability, and maintainability
* manual-first AI augmentation
* building small SRE or operations-supporting data applications

## Mandatory Career Layers

Every data engineering lab must also include:

* target job evidence
* analytical thinking
* clear explanation of findings
* stakeholder or manager communication
* recruiter-facing summary
* interview explanation
* personal reflection on accuracy, usefulness, and evidence quality

## Evidence Standard

Data engineering evidence should include safe sample data, schema notes, query text, Python scripts, result summaries, row counts, assumptions, limitations, data quality checks, and where relevant a small application or script.

Do not commit real company data, private user records, credentials, copied book text, or screenshots containing sensitive information.

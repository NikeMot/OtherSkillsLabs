# 04 — AWS

## Purpose

This folder contains labs for AWS administration, operations, reliability, monitoring, and cost-aware cloud support.

The goal is to build practical AWS skills that support SRE and platform engineering work.

## Topic Focus

Labs in this folder may cover:

* AWS account setup and safe lab hygiene
* IAM users, roles, groups, and policies
* VPC, subnets, routing, and security groups
* EC2 support and troubleshooting
* S3 storage, lifecycle, and access control
* RDS operational basics
* CloudWatch metrics, logs, and alarms
* Route 53 DNS scenarios
* load balancing and health checks
* backup and recovery
* tagging and cost awareness
* reliability evidence packs

## Mandatory Career Layers

Every AWS lab must also include:

* structured troubleshooting or design reasoning
* operational risk awareness
* manager or stakeholder communication
* recruiter-facing summary
* interview explanation
* reflection on production readiness

## Example Lab Folder Names

```text
lab-01-aws-account-and-iam-basics/
lab-02-vpc-and-networking-basics/
lab-03-ec2-operational-support/
lab-04-cloudwatch-monitoring-basics/
```

## Evidence Standard

AWS evidence must be sanitised. Do not commit account IDs, access keys, secret keys, private IP schemes from real environments, screenshots with sensitive details, or bill/account information.

# 04 — Cybersecurity

## Purpose

This folder contains defensive cybersecurity labs focused on practical blue-team and infrastructure-support skills.

The labs should help develop the ability to investigate, communicate, document, and explain security-relevant work in a professional way.

## Topic Focus

Labs in this folder may cover:

* identity and access review
* MFA and sign-in log analysis
* Windows and Linux security logs
* TLS and PKI troubleshooting
* endpoint security triage
* least privilege and local admin control
* hardening and baselines
* security reporting and escalation

## Mandatory Career Layers

Every cybersecurity lab must also include:

* structured thinking and evidence collection
* safe user or manager communication
* a recruiter-facing summary
* an interview or STAR story
* production security considerations
* personal reflection

## Example Lab Folder Names

```text
lab-01-user-access-review/
lab-02-tls-certificate-troubleshooting/
lab-03-security-log-review/
lab-04-endpoint-alert-triage/
```

## Evidence Standard

Cybersecurity evidence must be safe and sanitised. Do not commit real usernames, tenant IDs, IP addresses, company data, screenshots with private details, or live credentials.

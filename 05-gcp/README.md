# 05 — Google Cloud Platform

## Purpose

This folder contains labs for Google Cloud Platform administration, operations, reliability, monitoring, and cost-aware cloud support.

The goal is to build practical GCP skills that complement AWS and support SRE and platform engineering work.

## Topic Focus

Labs in this folder may cover:

* GCP projects and safe lab hygiene
* IAM principals, roles, and policies
* VPC networks, subnets, firewalls, and routes
* Compute Engine support and troubleshooting
* Cloud Storage access, lifecycle, and controls
* Cloud SQL operational basics
* Cloud Monitoring metrics, logs, and alerts
* Cloud DNS scenarios
* load balancing and health checks
* backup and recovery
* labels and cost awareness
* reliability evidence packs

## Mandatory Career Layers

Every GCP lab must also include:

* structured troubleshooting or design reasoning
* operational risk awareness
* manager or stakeholder communication
* recruiter-facing summary
* interview explanation
* reflection on production readiness

## Example Lab Folder Names

```text
lab-01-gcp-project-and-iam-basics/
lab-02-vpc-and-firewall-basics/
lab-03-compute-engine-operational-support/
lab-04-cloud-monitoring-basics/
```

## Evidence Standard

GCP evidence must be sanitised. Do not commit project IDs from real environments, service account keys, private credentials, screenshots with sensitive details, or bill/account information.

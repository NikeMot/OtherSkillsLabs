# Commit Strategy

## Purpose

Commits should make the repository easy to review, explain, and use as career evidence.

## Rules

* Keep commits small and logical.
* Use clear commit messages.
* Do not commit secrets or private data.
* Review changes before committing.
* Separate structure, documentation, and lab evidence when practical.

## Good Commit Messages

```text
Create initial repository structure
Add cybersecurity topic overview
Document lab output template
Add AI ticket triage lab outline
```

## Poor Commit Messages

```text
stuff
updates
final
work
misc
```

## Recommended Commit Types

| Type | Example |
| ---- | ------- |
| Structure | Create topic folder structure |
| Documentation | Document lab template |
| Lab evidence | Add Mac access troubleshooting lab |
| Reflection | Add career leverage notes |
| Cleanup | Fix formatting in lab write-up |

## Production Relevance

Readable commit history helps with review, rollback, troubleshooting, audit trails, and professional presentation.
